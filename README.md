# 실시간 참여형 빙고 게임 (Class Game Studio)

선생님과 학생이 각자의 기기로 접속해 실시간으로 함께 진행하는 수업용 빙고 게임 모음입니다. Firebase를 통해 학생 화면이 즉시 동기화되며, 별도의 설치 없이 웹 브라우저만으로 작동합니다.

## 주요 기능

- **두 가지 게임 모드**
  - 그림보고 랜덤 빙고: 선생님이 그림을 출제하면 학생이 빙고판에서 같은 단어를 찾아 클릭합니다.
  - 문제읽고 랜덤 빙고: 선생님이 텍스트 문제를 출제하면 학생이 빙고판에서 정답 그림을 클릭합니다. 문제·정답은 엑셀(.xlsx/.xls/.csv)로 일괄 등록합니다.
- **실시간 동기화**: 출제·정답 처리·우승자 판정이 모든 참여자 화면에 즉시 반영됩니다.
- **참여자별 무작위 빙고판**: 게임 시작 시 학생마다 서로 다른 빙고판이 자동 생성됩니다.
- **빙고판 크기 및 우승 조건 설정**: 3×3 / 4×4 / 5×5 선택, 우승 빙고 수(가로·세로·대각선 합산) 지정.
- **이미지 업로드**: 파일명이 그대로 빙고 단어로 사용됩니다.
- **선생님 단독 입장 잠금**: 한 방에 선생님은 한 명만 입장할 수 있도록 제어합니다.
- **방 분리**: 주소에 `?room=방이름`을 붙여 여러 학급을 동시에 운영할 수 있습니다.
- **우승 연출 및 효과음**: 색종이 애니메이션과 Web Audio 기반 효과음을 제공합니다.

## 실행 방법

1. Firebase 프로젝트를 준비하고 Realtime Database와 Storage를 활성화한 뒤, `firebase_config.js`의 설정값을 본인 프로젝트 정보로 교체합니다.
2. 정적 파일이므로 별도의 빌드 과정이 없습니다. 다음 중 한 가지 방법으로 제공합니다.
   - 정적 호스팅(예: Firebase Hosting, Vercel, GitHub Pages)에 배포합니다.
   - 또는 로컬에서 간단한 웹 서버로 실행합니다.

     ```bash
     # 예시 — 프로젝트 폴더에서
     python -m http.server 8000
     ```

     브라우저에서 `http://localhost:8000` 으로 접속합니다.
3. 시작 화면(`index.html`)에서 진행할 게임을 선택합니다.
4. 선생님은 선생님으로 입장해 그림(및 엑셀 문제)을 업로드하고 게임을 시작하며, 학생은 공유 링크로 접속해 닉네임을 입력하고 참여합니다.

> 참고: ES 모듈을 사용하므로 `file://`로 직접 열면 동작하지 않습니다. 반드시 웹 서버나 호스팅을 통해 접속해야 합니다.
> 문제읽고 랜덤 빙고 모드의 기본 선생님 비밀번호는 `123456`이며, 필요 시 `quiz_game.html`의 설정값을 수정합니다.

## 기술 스택

- HTML · CSS · JavaScript (Vanilla, ES Modules)
- Firebase Realtime Database (실시간 상태 동기화)
- Firebase Storage (이미지 저장)
- SheetJS(xlsx) — 엑셀 문제 가져오기
- Web Audio API — 효과음
